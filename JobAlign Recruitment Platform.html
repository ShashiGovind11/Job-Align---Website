Backend Structure:

Database Models:

  • User:
    id (primary key)
    email
    password (hashed)
    user_type ('job_seeker', 'employer', 'recruiter')
    company_name (for employer/recruiter)
    first_name, last_name (for job seeker and contact person)
    ... other profile details

  • Job:
    id
    title
    description
    requirements
    location
    salary_range
    employer_id (foreign key to User)
    recruiter_id (if posted by a recruiter, foreign key to User)
    created_at
    expires_at (optional)
    is_active

  • Application:
    id
    job_id
    job_seeker_id (foreign key to User)
    status (applied, reviewed, shortlisted, rejected, etc.)
    applied_at
    pre_screening_answers (could be a JSON field or a separate table)

  • PreScreeningQuestion:
    id
    job_id (each job can have its own set of questions)
    question_text
    question_type (text, multiple choice, etc.)

  • PreScreeningAnswer:
    id
    application_id
    question_id
    answer

APIs:

  • Auth: /api/auth/signup, /api/auth/login
  • Jobs: /api/jobs (GET, POST, PUT, DELETE)
  • Applications: /api/applications (POST to apply, GET to view applications by the user or by job)
  • PreScreening: /api/jobs/:jobId/questions (GET), /api/applications/:applicationId/answers (POST)

Backend Implementation Plan

The complete backend system will include the following components:

Database Schema (PostgreSQL)

-- Users Table
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    user_type VARCHAR(20) NOT NULL CHECK (user_type IN ('job_seeker', 'employer', 'recruiter')),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Job Seekers Profile
CREATE TABLE job_seekers (
    user_id INT PRIMARY KEY REFERENCES users(id),
    first_name VARCHAR(100) NOT NULL,
    last_name VARCHAR(100) NOT NULL,
    resume_url VARCHAR(255),
    skills TEXT[],
    preferred_locations VARCHAR(255)[],
    desired_salary NUMERIC
);

-- Employers Profile
CREATE TABLE employers (
    user_id INT PRIMARY KEY REFERENCES users(id),
    company_name VARCHAR(255) NOT NULL,
    industry VARCHAR(100),
    company_size VARCHAR(50),
    contact_person VARCHAR(255),
    phone VARCHAR(50)
);

-- Recruiters Profile
CREATE TABLE recruiters (
    user_id INT PRIMARY KEY REFERENCES users(id),
    agency_name VARCHAR(255) NOT NULL,
    specialization VARCHAR(100),
    years_experience SMALLINT,
    contact_person VARCHAR(255)
);

-- Jobs Table
CREATE TABLE jobs (
    id SERIAL PRIMARY KEY,
    employer_id INT REFERENCES users(id),
    recruiter_id INT REFERENCES users(id),
    title VARCHAR(255) NOT NULL,
    description TEXT NOT NULL,
    requirements TEXT NOT NULL,
    location VARCHAR(255),
    salary_range VARCHAR(100),
    employment_type VARCHAR(50),
    status VARCHAR(20) DEFAULT 'active' CHECK (status IN ('active', 'filled', 'closed')),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    expires_at TIMESTAMP
);

-- Applications Table
CREATE TABLE applications (
    id SERIAL PRIMARY KEY,
    job_id INT REFERENCES jobs(id),
    job_seeker_id INT REFERENCES users(id),
    status VARCHAR(20) DEFAULT 'submitted' CHECK (status IN ('submitted', 'reviewed', 'shortlisted', 'rejected', 'hired')),
    applied_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Pre-Screening Questions
CREATE TABLE screening_questions (
    id SERIAL PRIMARY KEY,
    job_id INT REFERENCES jobs(id),
    question TEXT NOT NULL,
    question_type VARCHAR(20) DEFAULT 'text' CHECK (question_type IN ('text', 'multiple_choice', 'boolean'))
);

-- Pre-Screening Answers
CREATE TABLE screening_answers (
    id SERIAL PRIMARY KEY,
    application_id INT REFERENCES applications(id),
    question_id INT REFERENCES screening_questions(id),
    answer TEXT
);

        const observer = new IntersectionObserver((entries) => {
            entries.forEach(entry => {
                if (entry.isIntersecting) {
                    entry.target.style.transform = 'translateY(0)';
                    entry.target.style.opacity = '1';
                }
            });
        }, { threshold: 0.1 });
